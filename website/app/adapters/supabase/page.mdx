# Supabase Postgres Adapter

The `tracelet_supabase` package provides a highly optimized, drop-in adapter to sync your background locations directly to a Supabase Postgres database using bulk inserts.

import { Steps, Callout } from 'nextra/components'

## Installation

Add the adapter to your `pubspec.yaml`:

```bash
flutter pub add tracelet_supabase
```

<Callout type="info">
  This package automatically depends on `supabase_flutter`. You must initialize the Supabase client before using this adapter.
</Callout>

## Setup & Configuration

<Steps>
### Database Schema
First, create a table in your Supabase Postgres database to receive the locations. 

```sql
CREATE TABLE public.locations (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id uuid REFERENCES auth.users NOT NULL,
  latitude double precision NOT NULL,
  longitude double precision NOT NULL,
  accuracy double precision,
  altitude double precision,
  speed double precision,
  heading double precision,
  timestamp timestamp with time zone NOT NULL,
  is_moving boolean,
  activity_type text,
  battery_level double precision,
  created_at timestamp with time zone DEFAULT now()
);

-- Enable RLS
ALTER TABLE public.locations ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users can insert their own locations" ON public.locations
  FOR INSERT WITH CHECK (auth.uid() = user_id);
```

### Initialize Supabase
Ensure Supabase is initialized in your `main.dart`:

```dart
import 'package:supabase_flutter/supabase_flutter.dart';

void main() async {
  WidgetsFlutterBinding.ensureInitialized();
  await Supabase.initialize(
    url: 'https://xyzcompany.supabase.co',
    anonKey: 'public-anon-key',
  );
  runApp(const MyApp());
}
```

### Configure the Adapter
When initializing Tracelet, mount the Supabase Adapter instead of using the standard HTTP Sync engine.

```dart
import 'package:tracelet_supabase/tracelet_supabase.dart';
import 'package:tracelet/tracelet.dart' as tl;

// 1. Create the adapter
final supabaseAdapter = TraceletSupabase(
  tableName: 'locations', // The Postgres table name
  batchSize: 500, // Number of rows per bulk insert
  autoSync: true, // Automatically listen to Tracelet's SQLite DB and flush
  
  // Optional: Transform the JSON before inserting to match your Postgres schema exactly
  transformPayload: (Map<String, dynamic> locationJson) {
    return {
      'user_id': Supabase.instance.client.auth.currentUser?.id,
      'latitude': locationJson['coords']['latitude'],
      'longitude': locationJson['coords']['longitude'],
      'accuracy': locationJson['coords']['accuracy'],
      'timestamp': locationJson['timestamp'],
      'is_moving': locationJson['is_moving'],
    };
  }
);

// 2. Start the adapter
await supabaseAdapter.start();

// 3. Start Tracelet
await tl.Tracelet.ready(tl.Config.balanced());
await tl.Tracelet.start();
```
</Steps>

## Performance & Delta Compression

Unlike Firebase which requires billing per-document-write, Supabase uses Postgres. The `tracelet_supabase` adapter utilizes PostgREST's bulk-insert capability (`.insert(List<Map>)`). 

If you queue 500 locations while driving offline, the adapter will send all 500 points in **a single HTTP request** and a **single Postgres transaction**. 

Because of this efficiency, Supabase is highly recommended for high-frequency tracking (e.g., 1 point every 10 meters).
